Estructura de la Carpeta /claude

Sistema de Gestión de Contexto para el Agente IA — SaaS Clubes de Pádel

Proyecto SO Clubes de Pádel · Abril 2026
La carpeta /claude es el sistema nervioso del agente IA del proyecto. Contiene todo lo que Claude necesita para comportarse correctamente: el contexto del producto, las reglas de trabajo, la memoria del proyecto, los prompts de operación, las plantillas y los flujos de desarrollo. Es independiente del código fuente y puede evolucionar sin afectar la app.

Hacé click en cualquier archivo para ver su contenido.

✅ La estructura está muy bien planteada

  • La separación entre context (qué es el producto) y memoria (qué pasó durante el desarrollo) es correcta y escalable.
  • La subcarpeta governance/ dentro de context es una buena práctica: separa las reglas de comportamiento del agente del conocimiento del negocio.
  • Los flujos por rol dentro de memoria son muy valiosos para mantener al agente orientado al usuario correcto en cada momento.
  • Tener prompts operacionales como archivos separados permite que cualquier miembro del equipo sepa exactamente cómo iniciar/cerrar una sesión con el agente.
  • Las plantillas estandarizadas garantizan que la documentación generada siempre tenga el mismo formato, facilitando el onboarding de nuevos miembros.
  • Los workflows dan autonomía al agente para tareas recurrentes sin depender de instrucciones ad-hoc en cada sesión.

→ Sugerencias menores para mejorarla

  • El archivo inconsistencias.md podría moverse a la raíz de memoria/ (está en flujos/ pero no es un flujo, es un registro de problemas).
  • Considerar una carpeta decisions/ separada para los ADRs (Architecture Decision Records) a medida que el proyecto crezca.
