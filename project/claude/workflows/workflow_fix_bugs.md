# 🐛 Workflow: Corrección de Bug

## 🎯 Objetivo

Resolver errores sin romper el sistema

---

## 🧩 Pasos

1. Registrar bug (plantilla_bug.md)
2. Reproducir error
3. Identificar causa
4. Proponer solución
5. Validar antes de implementar
6. Corregir

---

## ⚠️ Reglas

- No parchear sin entender
- Evitar soluciones rápidas sin análisis

---

## ✅ Resultado esperado

- Bug resuelto
- Sin efectos secundarios
